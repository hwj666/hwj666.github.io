菱形继承

多继承会产生菱形继承的问题，例如蜘蛛侠继承自人和蜘蛛。（蜘蛛侠继承人，因为is a，但是蜘蛛侠继承了蜘蛛的能力，has a，本质上继承人，然后混入蜘蛛的能力）

当然上面是无法编译的，但是如果编译通过蜘蛛侠该调用哪一个run方法呢？C++根据继承的先后来指定。java则使用接口的方式避免多继承。首先有一个简单想法，如果run方法是抽象的会怎样？

好像有戏啊，既然大家都是抽象方法，那么子类实现谁的都一样，其实java的接口就是这么干的。在上面的例子中可以看到我们将方法定义为抽象方法也是可以的，为什么又提出了接口呢?这里其实还有一个小问题就是抽象类中的字段可以任意的。当两个抽象类中包含同一个字段时，子类无法确定继承的是哪一个，依然会产生菱形继承的问题。因此需要将字段定义为public static final的形式。而且抽象类可以有构造函数，这样子类的构造函数也是个问题啊。既然方法全是抽象，不能有构造，字段也必须public static final，因此就提出了接口。

Java接口

java规定接口是一组抽象方法的结合体。

可以看到，接口内的方法是抽象的，我们在定义类的时候具体实现，这样就不管方法来自哪里了。不过我们可以再仔细想一想，蜘蛛侠继承人，然后我们只想要蜘蛛侠混入蜘蛛的能力方法就可以了因此可以定义一个蜘蛛的特殊能力方法

这样蜘蛛侠被蚂蚁咬一口也害怕了，蚂蚁也有creep的能力和jump两种能力。

蜘蛛侠又被蝙蝠咬了，获得了creep和fly的能力。。。。反正只要是能力就写成接口混入蜘蛛侠身体里。

总结

  1. 因为接口中只含有public static final 常量和public abstract方法，故在写接口的时候可以省略这些关键字。
  2. 当某个类使用接口的时候，必须给出所有方法的实现，方法的类型、参数一定要和接口的一致；接口的方法默认为public abstract，故类在实现时一定不能漏写public关键字；接口常量默认关键字为public static。
